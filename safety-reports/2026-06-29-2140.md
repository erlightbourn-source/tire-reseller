## Fixed & committed

- **[correctness, high]** Listing edit crashed with a runtime `ReferenceError`.
  `app/api/listings/[id]/route.js` — commit `4827a61`.
  The `PATCH` handler calls `deriveListingColumns(...)` (line ~68) to keep the
  denormalized `widthMm/aspectRatio/rimDiameter/treadDepth32/perTireCents` columns in
  sync, but the function was never imported in this file (it is imported in
  `app/api/listings/route.js` and `app/api/listings/bulk/route.js`). Any owner edit that
  changed size, treadDepth, price, or quantity threw a `ReferenceError`, returned a 500,
  and the edit silently failed; the denormalized columns also drifted, which degrades the
  browse size/price filters that read them. Runtime-only error, so it was invisible to
  `next build`. Fix: added `import { deriveListingColumns } from "@/lib/tiresize";`,
  matching the other two listing routes. One-line, behavior-preserving.
  Impact: restores a core seller workflow (editing a listing's size/price/quantity).
